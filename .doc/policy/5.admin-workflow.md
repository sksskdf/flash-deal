## 상품 관리

### 상품 등록 플로우
```
1. 상품 정보 입력
   ├→ 기본 정보 (제목, 설명, 카테고리)
   ├→ 가격 정보 (원가, 할인가, 통화)
   ├→ 이미지 업로드 (메인, 상세, 옵션)
   └→ 스펙 정보 (동적 속성)

2. 딜 설정
   ├→ 딜 타입 선택 (Flash/Time/Group)
   ├→ 일정 설정 (시작/종료 시간)
   ├→ 재고 설정 (총 수량, 안전 재고)
   └→ 정책 설정 (구매 제한, 예약 시간)

3. 검토 및 승인
   ├→ 정보 검증
   ├→ 이미지 품질 확인
   ├→ 가격 정책 검토
   └→ 승인/반려 결정

4. 게시
   ├→ 상품 활성화
   ├→ 검색 인덱스 업데이트
   ├→ 캐시 무효화
   └→ 알림 발송 (관리자/사용자)
```

### 상품 수정 플로우
```
1. 수정 요청
   ├→ 수정 사유 입력
   ├→ 변경 내용 명시
   └→ 승인자 지정

2. 수정 처리
   ├→ 기존 정보 백업
   ├→ 변경 내용 적용
   ├→ 연관 데이터 업데이트
   └→ 버전 관리

3. 검증 및 배포
   ├→ 변경 사항 검증
   ├→ A/B 테스트 (필요시)
   ├→ 단계적 배포
   └→ 롤백 계획 수립
```

### 상품 삭제/비활성화
```
1. 삭제 요청
   ├→ 삭제 사유 입력
   ├→ 연관 주문 확인
   └→ 영향도 분석

2. 삭제 처리
   ├→ 활성 주문 확인
   ├→ 재고 정리
   ├→ 연관 데이터 정리
   └→ 아카이브 처리

3. 후속 처리
   ├→ 사용자 알림
   ├→ 대체 상품 제안
   └→ 데이터 백업
```

## 주문 관리

### 주문 처리 플로우
```
1. 주문 접수
   ├→ 주문 정보 확인
   ├→ 결제 상태 확인
   ├→ 재고 확인
   └→ 배송 가능 여부 확인

2. 주문 검증
   ├→ 고객 정보 검증
   ├→ 배송지 검증
   ├→ 결제 정보 검증
   └→ 상품 가용성 확인

3. 주문 처리
   ├→ 재고 확보
   ├→ 배송 라벨 생성
   ├→ 택배사 연동
   └→ 고객 알림 발송

4. 배송 추적
   ├→ 택배사 API 연동
   ├→ 실시간 추적 정보
   ├→ 고객 알림
   └→ 예외 상황 처리

```
1. 취소 요청 접수
   ├→ 취소 사유 확인

   ├→ 재고 복구
   ├→ 결제 취소
   ├→ 배송 취소 (필요시)

   ├→ 결제 수단별 환불
